# 演唱会歌词标记工具 — 设计文档

## 概述

一个基于 Web 的纯前端单页应用，用于：加载演唱会音频 → 边听边通过按钮标记每首歌的开始时间 → 配合已检索的歌词 → 导出带偏移时间戳的 LRC 文件。

## 技术选型

| 项 | 选择 | 理由 |
|---|---|---|
| 架构 | 纯前端单页应用（SPA） | 零依赖，双击 HTML 即用 |
| 音频 | Web Audio API | 浏览器原生播放控制 |
| 歌词数据源 | LRCLIB API（lrclib.net） | 免费、开放、支持 CORS |
| 数据存储 | localStorage | 每次操作自动保存，不丢进度 |
| 导出方式 | Blob URL 下载 | 无需后端 |

单个 HTML 文件，依赖通过 CDN 加载，零后端。

├──────────────┬─────────────────────────────┬────────────────────┤
│  演唱会音频   │  播放控制                    │  歌词区            │
│              │  ⏮ ◀ ▶ ▶ ⏭                 │                    │

│  📁 北京站   ├─────────────────────────────┤  ▶ 等待在这雪山路  │
│              │  歌曲列表                    │  听寒风呼啸依旧    │

三栏布局：
- **左侧**：演唱会音频文件列表（多个 MP3 切换，支持添加/删除音频文件）
- **中间上方**：音频播放控制（播放/暂停、快进/快退、倍速）+ 标记按钮
- **中间下方**：歌曲列表（序号、歌名、开始时间、歌词状态、每首歌有"导入"/操作按钮）
- **右侧**：当前歌曲歌词预览（完整显示，当前行高亮标记，不自动滚动）

## 数据模型

```json
{
  "id": "project-unique-id",
  "name": "演唱会名称",
  "artist": "歌手名（全局默认）",
  "mp3_name": "文件名.mp3",
  "songs": [
    {
      "id": 1,
      "title": "歌名",
      "artist_override": null,
      "start_time": 0,
      "lyrics_status": "confirmed",
      "lyrics_plain": "纯文本歌词",
      "lyrics_lrc": "[00:00.00]歌词行1\n[00:04.50]歌词行2",
      "lyrics_source": "lrclib"
    }
  ]
}
```

- `artist_override`：单首歌歌手名，为 null 时使用全局 artist
- `lyrics_status`：`pending` / `confirmed` / `not_found` / `manual`

## 核心流程

### 1. 新建项目
填写演唱会名称 + 全局歌手名 → 创建空项目。

### 2. 导入歌单 + 歌词检索（两步弹窗）

**流程：**

**Step 1 — 导入歌单（弹窗）：**
- 点击顶栏"导入歌单"，弹出导入弹窗
- 在文本框中粘贴歌单（支持 `1. 歌名` / `01 - 歌名` / `歌名` 等格式）
- 点击"解析"，自动提取歌名列表
- 预览解析结果，歌名可编辑修改
- 点击"下一步"进入 Step 2

**Step 2 — 歌词检索（弹窗内左右布局）：**
- 弹窗内左右分栏：左侧为歌曲检索结果列表，右侧为歌词预览区
- 左侧逐首调用 LRCLIB API 检索，显示进度条和每首歌的检索状态
- 每首歌显示匹配歌手、歌词类型（同步/纯文本/未找到）
- 点击[查看]或点击歌曲行，右侧显示该首歌词预览，可编辑修改
- 匹配不对可改歌手名重搜
- 点击"完成"后，歌词数据保存到项目，弹窗关闭

### 3. 载入音频 + 标记时间轴

- 选择 MP3 文件载入播放器
- 播放音频，听到下一首歌开始时，点击"标记"按钮
- 记录当前播放时间作为该歌的 start_time
- 歌曲列表中实时更新时间和状态

歌词数据在第二步已完成，标记时不需要再处理歌词。

### 4. 歌词预览

- 右侧歌词区显示当前歌曲的完整歌词
- 当前演唱行高亮标记（行首加指示符）
- 不自动滚动，整篇可见

### 5. 导出 LRC

导出时做时间偏移：

```
[start_time + 歌词相对时间] 歌词内容
```

例：西海情歌标记开始时间 09:15，其 LRC 原始 `[00:04.50]从此就丢了温柔` → 导出 `[09:19.50]从此就丢了温柔`

每首歌生成一个 .lrc 文件，打包下载。

## 弹窗汇总

| 弹窗 | 触发 | 说明 |
|---|---|---|
| 导入歌单 | 顶栏"导入歌单" | 单步：粘贴歌单 → 解析 → 确认后关闭 |
| 歌词检索（右侧面板） | 上一步关闭后自动切换 | 进度条 + 结果列表 + 查看/编辑/重搜 + 确认完成 |
| 项目管理 | 顶栏"项目管理" | 查看/切换/删除项目 |

## 数据持久化

- localStorage 自动保存
- 关键操作（导入歌单、歌词确认、标记时间）后即时写入
- 支持多项目切换

## 依赖

| 名称 | 用途 | 加载方式 |
|---|---|---|
| LRCLIB API | 歌词检索 | 直接 fetch，支持 CORS |

不需要任何 npm 包或后端服务。
