# 演唱会歌词标记工具 — 设计方案

## 概述

一个基于 Web 的本地工具，用于：加载演唱会 MP3 → 边听边通过按钮标记每首歌的开始时间 → 自动检索歌词 → 用户核对确认 → 导出 LRC 文件。

---

## 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 后端 | Python 3 + Flask | 轻量，音频处理库丰富 |
| 前端 | HTML + vanilla JS + WaveSurfer.js | 波形可视化 + 音频播放一体 |
| 数据 | JSON 文件存储 | 演唱会项目数据量小，无需数据库 |
| 歌词来源 | LRCLIB API (lrclib.net) | 免费、开放、支持同步歌词(LRC) |

---

## 使用流程

```
① 新建项目 → 输入演唱会名称 + 歌手名
② 选择 MP3 文件 → 加载到播放器
③ 粘贴歌单（每行一首歌名）→ 解析为歌曲列表
④ 按"播放"开始听 → 每听到下一首歌开始时，点击"标记下一首"按钮
⑤ 所有歌标记完毕后 → 点击"检索歌词"，工具自动搜索每首歌的歌词
⑥ 逐个核对歌词 → 可编辑修改 → 确认
⑦ 导出 → 每首歌生成一个 .lrc 文件，打包到项目目录
```

## 核心功能模块

### 1. 音频播放 + 波形 + 标记

- 使用 **WaveSurfer.js** 渲染波形图，支持点击跳转、缩放
- 播放/暂停/快进快退 按钮
- **"标记下一首"按钮**（大按钮，方便点击）：点击后记录当前播放时间，自动推进到歌单下一首
- 标记列表实时显示在侧边栏，可删除/手动编辑时间
- 支持拖拽 MP3 文件或点击选择

### 2. 歌单解析

- 文本框粘贴歌单，每行一首
- 自动按行分割，去除空行和序号（如 "1. "、"01 - " 等前缀）
- 支持格式：`歌名` 或 `序号. 歌名` 或 `序号 歌名`

### 3. 歌词检索（关键设计）

- 优先使用 **LRCLIB API**：`GET /api/get?track_name=歌名&artist_name=歌手`
- 返回数据包含：
  - `plainLyrics`：纯文本歌词
  - `syncedLyrics`：带时间戳的 LRC 格式歌词（如果有）
- 如果 LRCLIB 有同步歌词 → 直接使用，时间偏移量自动对齐到歌曲开始时间
- 如果只有纯文本 → 保留纯文本，导出时生成简单 LRC（仅第一行带时间戳）
- 如果完全搜不到 → 标记为"未找到"，由用户手动输入

### 4. 歌词核对界面

- 表格展示所有歌曲：序号、歌名、开始时间、歌词状态（有/无/待确认）
- 点击某首歌 → 弹出歌词编辑窗口
- 显示检索到的歌词，用户可编辑、替换、清空
- 支持手动粘贴歌词
- 确认后状态变为 "confirmed"

### 5. LRC 导出

- 每首歌生成一个 `.lrc` 文件，文件名 = `{序号} - {歌名}.lrc`
- 如果有同步歌词：将原 LRC 的时间戳按歌曲开始时间偏移
- 如果只有纯文本：生成 `[mm:ss.xx]歌词行` 格式
- 一键导出到项目目录下的 `lrc/` 文件夹

## 歌词检索的容错策略

1. 先用 `歌名 + 歌手全名` 搜索
2. 如果无结果，尝试 `歌名 + 歌手简称`（如"刀郎"→"刀郎"）
3. 如果仍无结果，仅用 `歌名` 搜索，取第一个结果
4. 全部失败则标记为"需手动输入"

2. 粘贴测试歌单
3. 播放并依次标记每首歌的开始时间
